External CSS Document

CSS stands for cascading stylesheet. It applies style in the form of colors, fonts, spacing, and much more. We have already seen some CSS styling in the inline styles we did. However, this is not how CSS is typically done. CSS typically has its own sheet, a stylesheet, which defines the style for any HTML element you wish.

To begin, we will need to create a new file in this project. This file is commonly called style.css. Once we have created a CSS stylesheet, we must LINK IT to the HTML documents. See the head section above to link CSS Stylesheet.

CSS Syntax

CSS is written similar to Java, in that each statement must end in a semi-colon. Brackets are used to denote a group of styles for one element. Each block of style first selects an HTML element. This is called the selector. ANY HTML element can be a selector. The style below selects all paragraph elements and sets its color to blue.


    p{

        color: blue;

    }

Hierarchy of CSS

Hierarchy means, if two different styles attempt to style the same element, hierarchy determines which is shown. In CSS, and in most languages, hierarchy is determined by what gets processed last. The head section of pages gets processed first, then the HTML gets loaded and styles processed. So any inline CSS will override any CSS in an external document.

Review Known Styles

color changes the foreground color. background-color changes the background color. font-family changes the font style and font-size changes the size of the font.

Custom Selectors

Sometimes we want the same styling on two different HTML elements We can select more than one element by separating each with a comma in the selector of the CSS. The code below selects both the paragraph element and the un-ordered list elements.

Selecting Multiple Elements


        p, ul{

            color: blue;

        }
